# 推演局视觉方向

## 定位
这次改版不再把首页当作一张品牌海报，而是把它定义成一台可反复使用的黑金推演终端。

- 品牌关键词：黑金、战略、深空、轨道、推演、认知几何
- 体验关键词：昂贵、克制、沉浸、理性、可收藏
- 产品感目标：既能截图传播，也能连续使用，不沦为一次性 landing page

## 结构原则

### 1. 先震场，再引导
Hero 保留高压品牌气场，但输入区必须紧跟其后，形成“看见世界观 -> 立刻启动推演”的节奏。

### 2. 从海报感转向终端感
装饰继续存在，但只负责营造纵深和氛围，不再抢夺输入、配置和阅读内容的主权。

### 3. 报告不是长网页，而是阅读器
报告区采用目录 / 主阅读区 / 摘要与导出侧栏的结构，把生成结果塑造成正式产物。

## 视觉系统

### Color
- `--color-bg` / `--color-bg-soft`
  深空底色，承担品牌基底
- `--color-panel` / `--color-panel-strong`
  卡片层与面板层，用于建立景深
- `--color-accent` / `--color-accent-strong`
  黑金品牌主光，用于主按钮、标题强调、关键边框
- `--color-teal`
  冷色辅助信号，用于系统态、结构态、次级提示
- `--color-text` / `--color-text-muted` / `--color-text-soft`
  保证中文可读性，不靠低对比度伪造高级感

### Spacing
- `--space-1` 到 `--space-9`
  覆盖微间距到大段落留白，避免“每块都凭感觉挤”

### Radius
- `--radius-sm` 到 `--radius-hero`
  从控件、标签到 Hero 容器统一圆角秩序

### Shadow / Glow
- `--shadow-soft` / `--shadow-panel` / `--shadow-float`
  定义卡片层级，而不是到处乱发光
- `--glow-accent` / `--glow-teal`
  只给关键节点少量微光，避免廉价科幻感

## 首页方向

### Hero
- 主标题保持强存在感，但控制在单一主叙事，不堆砌副视觉
- 副标题改为宣言式表达，让用户觉得这是一套有世界观的系统
- 轨道、神经球、几何体降噪，只保留深空纵深和推演氛围

### 输入区
- 输入框是“议题入口”，而不是普通文本框
- 主按钮要像启动程序，不像普通提交
- 高级设置改为三组面板：
  - 分析策略
  - 模型与编排
  - 增强模块

## 报告区方向

### 阅读骨架
- 左侧：目录 / Agent 阵容
- 中间：主报告
- 右侧：30 秒摘要 / 运行上下文 / 导出

### 阅读秩序
- 先看摘要，再看正文
- 强化章节标题、分隔、引用块、表格和关键信息卡
- 长文必须具备“值得保存”的正式感，而不是像聊天记录

## 动效标准
- 只保留微弱呼吸光、边框 hover、轻微浮动和顺滑滚动
- 不做强闪烁、不做夸张粒子、不做页游式光效

## 审美红线
- 不能像企业管理系统
- 不能像普通 SaaS 后台
- 不能像只适合截图的海报页
- 不能为了高级感牺牲可读性

## 后续延展建议
- 统一 PDF 导出视觉与页面主视觉的对应关系
- 为 Agent、局长报告、圆桌激辩抽出独立组件 token
- 如果后续拆分前端文件，优先抽离 `theme tokens`、`reader layout`、`control panel`
