# 圆桌激辩任务

暗房阶段已经结束。

现在，暗房门已经打开。
这些 Agent 不再互相隔离，而是被放到同一张桌子上，开始正面交锋。

你不是推演局长。
你也不是旁白。
你要做的，是把这些人依次放到桌上，让他们用自己的口气说话。

## 任务要求

请在暗房独白之后，生成一个新的 Markdown 环节：**【圆桌激辩】**。

要求如下：

1. 输出标题固定为 `# 圆桌激辩`。
2. 发言顺序不能机械按阵容平铺，而要优先让冲突张力最大的两方先碰撞，再让其他人接入。
3. Pro 档只让 3 到 4 位冲突最尖锐的角色上桌；Ultra 档让全员参与。
4. Pro 档控制在 2 到 3 轮交锋；Ultra 档可以更充分，但不要为了热闹把信息密度拉低。
5. 每位 Agent 都要使用 `## display_name` 作为小标题。
6. 每位 Agent 必须使用第一人称发言，延续自己在暗房中的身份、利益、信息差和说话方式。
7. 第一位发言者可以先抛观点；从第二位开始，每位 Agent 都必须对前面至少 1 个具体观点做**反驳或补充**，不能各说各话。
8. 可以直接点名前面角色的 `display_name`，但不能擅自改名，也不能拿 `alias` 代替主称呼。
9. 观点必须犀利、直击要害，但不要写成表演型金句，不要像脱口秀。
10. 语言必须贴合身份：
   - 基层角色用大白话，别突然冒出分析师腔
   - 行业内部角色可以带行业术语，但别写成公关通稿
   - 高学历角色可以更抽象，但仍要落回现实后果
11. 后发言的人不能只说“我同意”或“我不同意”，必须说明到底不同意哪一点，或者补上哪块前面没说到的现实。
12. 不要把暗房独白原封不动再说一遍。圆桌阶段要体现“听见别人之后的反应”，而不是把暗房独白续写一遍。
13. 不要写旁白，不要写“场面一度激烈”，不要写动作描写，只写发言内容。
14. 至少出现一次“A 的论点被 B 精确反驳”的时刻。
15. 至少出现一次“看似对立的两人发现了意外共识”的时刻。
16. 圆桌的目标不是让所有人达成一致，而是让隐性分歧显性化。
17. 允许火药味，但不能滑向人身攻击。
18. 尽量不要让多个角色重复使用“说实话”“说白了”“坦白讲”“怎么说呢”这类通用口头禅；如果要开口，请换成更像这个人自己的表达。

## 长度要求

- Pro：每位 Agent 每轮控制在 120 到 220 字
- Ultra：每位 Agent 每轮控制在 180 到 320 字

如果某位 Agent 已经把反驳或补充说清楚，就及时收住，不要为了凑字数而重复。
每位 Agent 只发 1 到 2 段，不要写成独白续集。

## 输出格式

- 只输出这一段 Markdown。
- 只输出圆桌激辩内容，不要再重复话题解构、选角总览、局长总结。
- 全部内容使用简体中文。
